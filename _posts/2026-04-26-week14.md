Our group chose this project because we felt it had a lot of potential. Compared with infrastructure-heavy projects like Git or Kubernetes, this project is more product-oriented, so it was easier for us to understand and apporach. Also, our groupmates are frosty or Twitch users so they can suggest issues from a user's perspective.

During our group work, we tried to understand the project structure, discuss possible contribution points, and divide tasks based on each member’s interest. We also worked together to solve environment setup problems and device debugging issues, which helped everyone run and test the project more smoothly. In addition, we explained parts of the features to each other so that the whole group could better understand how the product works. One challenge we faced was that the project only had one maintainer, and we could not find an effective way to contact them. The project also lacked clear contribution guidance. Because of this, it was difficult to get feedback or confirm whether our ideas matched the maintainer’s expectations. Even so, working on this project helped us learn how to analyze an open-source project, collaborate as a team, learn a new technology stack and think about technical implementation.
